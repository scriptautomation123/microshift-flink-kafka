# Podman + MINC Cluster Setup, Cleanup, and Reinstall Guide

This guide covers everything needed to run the MicroShift cluster on Ubuntu alongside Docker: installing Podman, installing MINC, accessing the cluster, configuring DNS, handling storage/PVCs, and fully cleaning up or reinstalling.

## Prerequisites

- Ubuntu 22.04+
- Docker may already be installed; Podman and Docker can coexist with forwarding rules
- `podman` (3.4+) or `docker` (20.10+)
- 8+ GB RAM (16+ GB recommended)
- 50+ GB disk
- Internet connectivity for package/image downloads

## Install Podman

Install the host packages used by MINC / MicroShift:

```bash
sudo apt-get update
sudo apt-get install -y \
  ca-certificates \
  conntrack \
  curl \
  iptables \
  jq \
  nftables \
  uidmap \
  podman
```

Validate:

```bash
podman info --format '{{.Host.NetworkBackend}} {{.Host.Security.Rootless}}'
```

## Run Podman Alongside Docker

If Docker is running, Podman needs explicit forwarding rules so the two networks can communicate.

**Automated path**:

```bash
sudo microshift.sh install-forwarding
```

**Manual path**:

```bash
sudo iptables -I DOCKER-USER 1 -i podman0 -j ACCEPT
sudo iptables -I DOCKER-USER 2 -o podman0 -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT
```

## Install MINC

```bash
curl -fsSL -o /tmp/minc https://github.com/minc-org/minc/releases/download/v0.1.0/minc-linux-amd64
sudo install -m 0755 /tmp/minc /usr/local/bin/minc
minc version
```

For arm64, use the matching release asset.

## Create the Cluster

### Automated workflow commands

`microshift.sh` now provides high-level flow commands that run the manual steps for you:

| Command | Purpose |
| --- | --- |
| `microshift.sh install-fresh` | First-time install: DNS, Docker forwarding, MINC create, kubeconfig, PVs |
| `microshift.sh reinstall-fresh --wipe-local-data` | Full teardown + fresh install |
| `microshift.sh delete-all --delete-pvcs --delete-pvs` | Delete cluster, PVCs, and PVs |
| `microshift.sh resume --check-pv` | Regenerate kubeconfig and validate after reboot |
| `microshift.sh status` | Show cluster, access, and storage status |

#### First-time install

```bash
sudo microshift.sh install-fresh
```

This runs:

1. Podman DNS configuration
2. Docker-to-Podman forwarding (only if Docker is active)
3. `minc create`
4. `minc generate-kubeconfig` + `oc config use-context microshift`
5. Local PV provisioning

#### Full reinstall (wipe everything)

```bash
sudo microshift.sh reinstall-fresh --wipe-local-data
```

This deletes PVCs, PVs, the MINC cluster, local host data, and then runs `install-fresh`.

#### Continue after reboot

```bash
microshift.sh resume --check-pv
```

#### Check status

```bash
microshift.sh status
```

### Manual cluster creation

If you prefer to run each step yourself:

```bash
minc config set provider podman
minc config set allow-rootless true
minc config view

minc delete || true
minc create
```

Validate:

```bash
minc status
minc list
```

Expected output shows `microshift` with ports:

```text
127.0.0.1:6443->6443/tcp, 127.0.0.1:9080->80/tcp, 127.0.0.1:9443->443/tcp
```

## Access the Cluster

### First-Time Setup

After `minc create`, generate the kubeconfig and switch context:

```bash
minc generate-kubeconfig
oc config use-context microshift
```

This only needs to be done once after cluster creation, or after deleting `~/.kube`.

### Do Not Use `oc login`

`oc login` uses OpenShift OAuth discovery at `/.well-known/oauth-authorization-server`. MINC MicroShift does not expose this endpoint, so `oc login` will fail with a 404 even though the API is healthy. Use the kubeconfig generated by MINC instead.

### Validate Access

```bash
export PATH="$HOME/.local/bin:$PATH"
oc whoami
oc whoami --show-server
oc get nodes -o wide
oc get pods -n openshift-operator-lifecycle-manager -o wide
ss -ltnp | grep -E ':(6443|9080|9443)\b' || true
```

Expected gates:

- ✓ `oc whoami` succeeds (typically `system:admin`)
- ✓ Node is `Ready`
- ✓ OLM pods (`catalog-operator`, `olm-operator`) running
- ✓ Listeners on 6443, 9080, 9443

## Configure Podman DNS

If Docker is running, Podman may not have working DNS unless explicitly configured.

**Automated path**:

```bash
sudo microshift.sh configure-dns --dns1 1.1.1.1 --dns2 8.8.8.8
```

**Manual path**:

```bash
sudo install -d -m 0755 /etc/containers
sudo tee /etc/containers/containers.conf >/dev/null <<'EOF'
[containers]
dns_servers = ["1.1.1.1", "8.8.8.8"]
EOF
```

Validate DNS:

```bash
sudo podman run --rm docker.io/library/alpine:3.20 cat /etc/resolv.conf
sudo podman run --rm docker.io/library/busybox:1.36 nslookup quay.io 1.1.1.1
```

Recreate the cluster after changing DNS or forwarding rules:

```bash
minc delete || true
minc create
minc generate-kubeconfig
oc config use-context microshift
```

## PVC and Storage Handling

### Check PVC State

```bash
oc get pv
oc get pvc -n kafka-dev
oc get pvc -n flink-dev
```

### Symptom: Pods Stay Pending

```text
pod has unbound immediate PersistentVolumeClaims
```

#### Automated path

```bash
microshift.sh provision-pv --kafka-namespace kafka-dev --flink-namespace flink-dev
```

#### Manual PV provisioning

The helper above creates static `hostPath` PVs with a pre-bound `claimRef`. To do the same thing manually for the default layout (3 Kafka brokers, 1 Flink JobManager, 3 Flink TaskManagers):

1. Create the host directories:

   ```bash
   sudo mkdir -p /var/lib/microshift-local-pv/kafka-dev/kafka-0
   sudo mkdir -p /var/lib/microshift-local-pv/kafka-dev/kafka-1
   sudo mkdir -p /var/lib/microshift-local-pv/kafka-dev/kafka-2
   sudo mkdir -p /var/lib/microshift-local-pv/flink-dev/jobmanager-0
   sudo mkdir -p /var/lib/microshift-local-pv/flink-dev/taskmanager-0
   sudo mkdir -p /var/lib/microshift-local-pv/flink-dev/taskmanager-1
   sudo mkdir -p /var/lib/microshift-local-pv/flink-dev/taskmanager-2
   ```

2. Apply the PV manifests. Here is the Kafka broker example:

   ```yaml
   apiVersion: v1
   kind: PersistentVolume
   metadata:
     name: local-pv-kafka-dev-kafka-0
     labels:
       app.kubernetes.io/managed-by: microshift-local-pv
   spec:
     capacity:
       storage: 50Gi
     volumeMode: Filesystem
     accessModes:
       - ReadWriteOnce
     persistentVolumeReclaimPolicy: Retain
     claimRef:
       namespace: kafka-dev
       name: data-kafka-0
     hostPath:
       path: /var/lib/microshift-local-pv/kafka-dev/kafka-0
       type: DirectoryOrCreate
   ```

   Create one PV per PVC expected by the StatefulSets. The default claim names are:

   - Kafka: `data-kafka-0`, `data-kafka-1`, `data-kafka-2`
   - Flink JobManager: `jobmanager-data-flink-jobmanager-0`
   - Flink TaskManagers: `taskmanager-data-flink-taskmanager-0`, `taskmanager-data-flink-taskmanager-1`, `taskmanager-data-flink-taskmanager-2`

   Apply each manifest:

   ```bash
   oc apply -f pv-kafka-0.yaml
   # ... repeat for each PV
   ```

Then verify:

```bash
oc get pv
oc get pvc -n kafka-dev
oc get pvc -n flink-dev
```

### Delete PVCs During Cleanup

When removing a namespace, PVCs must be deleted explicitly if you want to free the underlying local PV data:

```bash
oc delete pvc --all -n kafka-dev
oc delete pvc --all -n flink-dev
oc delete namespace kafka-dev flink-dev
```

For a full cluster teardown, see [Full Reinstall](#full-reinstall).

## Cluster Cleanup

### Automated cleanup

```bash
# Delete cluster, PVCs, and PVs
microshift.sh delete-all --delete-pvcs --delete-pvs

# Also wipe local host data
sudo microshift.sh delete-all --delete-pvcs --delete-pvs --wipe-local-data
```

### Manual cleanup

Keep MINC, remove workloads only:

```bash
# Remove Kafka and Flink workloads
oc delete statefulset,deployment -n kafka-dev
oc delete statefulset,deployment -n flink-dev

# Remove pods and namespaces
oc delete pod -n kafka-dev --all
oc delete pod -n flink-dev --all
oc delete ns kafka-dev flink-dev
```

Remove MINC cluster but keep configuration:

```bash
minc delete
minc list  # should be empty
```

## Full Reinstall

Use this when the cluster has stale references after a reboot, DNS changes, or when you want a completely fresh environment.

### Automated reinstall

```bash
sudo microshift.sh reinstall-fresh --wipe-local-data
```

### Manual reinstall

```bash
# 1. Delete workloads and free PVCs
oc delete pvc --all -n kafka-dev 2>/dev/null || true
oc delete pvc --all -n flink-dev 2>/dev/null || true
oc delete namespace kafka-dev flink-dev 2>/dev/null || true

# 2. Delete the MINC cluster
minc delete || true

# 3. Remove leftover container/network artifacts
sudo podman rm -f microshift 2>/dev/null || true
sudo podman network rm podman 2>/dev/null || true
sudo podman network create podman

# 4. Re-create the cluster
minc create

# 5. Regenerate kubeconfig
minc generate-kubeconfig
oc config use-context microshift

# 6. Validate
oc whoami
oc get nodes -o wide
```

## Troubleshooting

### `oc login` fails with 404

Use kubeconfig authentication instead:

```bash
minc generate-kubeconfig
oc config use-context microshift
```

### MINC create fails with rootless error

```bash
minc config set allow-rootless true
minc create
```

### OLM pods in `ImagePullBackOff`

Check Podman DNS and egress:

```bash
sudo podman run --rm docker.io/library/busybox:1.36 nslookup quay.io 1.1.1.1
```

On Docker coexistence hosts, verify forwarding rules allow `podman0` traffic through `DOCKER-USER`.

### Pods stay `Pending` with unbound PVCs

Provision local PVs:

```bash
microshift.sh provision-pv --kafka-namespace kafka-dev --flink-namespace flink-dev
```

### Stale cluster after host reboot

Use the automated reinstall:

```bash
sudo microshift.sh reinstall-fresh --wipe-local-data
```

Or run the [Full Reinstall](#full-reinstall) procedure manually.
